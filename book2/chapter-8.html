Flatland aesthetic

A blog post called “The Flat Design Era” has been going the rounds the last few days.

I’ve long been a fan of flat design; if you would give me a blank canvas to draw all UI elements for an OS the end result would probably resemble Metro more than iOS.

In any case, one problem with the “flat design” aesthetic is that it’s hard to make out what parts of the UI you can actually interact with.

When the only difference between a button and a background is the size of the container, things can get confusing.

A button should look like a button.

Text that can be interacted with should be styled differently than other text.

Links should have consistent colors.

Here’s a good example: Khoi Vinh‘s web site. Is that bold text or a link?

From an aesthetic perspective that page looks mighty fine. From a usability perspective, one could argue that links need to be visible colors. Another standpoint might argue that people will find out soon enough because of the hover state (an argument that no longer holds true in this touch device era),

These things might help a button look like a button: inset highlight/shadow, drop shadow, a background change, a text color change, a gradient, rounded corners, a border.

iOS buttons have all of these elements (they have been toned down a bit in iOS6, which I can only applaud) . Most Metro buttons only have a background and text change.

What’s best then? As always, it depends. Consistency is the most important thing.

If you really want to, you could make everything black, but then everything that is not clickable needs to be grey enough to see the difference.
